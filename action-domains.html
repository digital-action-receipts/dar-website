Where DAR applies

Action Domains

DAR is a universal evidence layer: it emits receipts after actions commit (non-blocking), and stores them in customer-controlled, immutable/append-only storage. These domains represent the highest-leverage places where disputes, audits, and investigations require durable proof.

Principle
DAR is not monitoring. DAR is not decision logic. DAR is not a system of record. DAR is evidence — issued at action time, held under customer custody, and verified later.

Quick map

Each domain below is a repeatable pattern: a high-stakes action occurs → a receipt is emitted → custody remains with the customer.

Non-repudiation Access decisions State & config changes Agent actions AI accountability Insurance & claims Audit continuity Disputes & investigations

When an action is later denied (“we didn’t do that”), DAR provides durable proof bound to actor, action, object, time, and context references.

Audit asks: “prove it.” DAR gives a portable evidence layer that remains valid even if the vendor’s database is unavailable or changed.

Disputes, investigations, and litigation support

When narratives diverge, receipts reduce ambiguity by anchoring a stable timeline of facts—what occurred, when, and under which references.

timeline chain of custody verification digital evidence

Domain pattern (reusable)

Most domains share the same mechanics. The only difference is the event taxonomy and what context references matter.

1) Action commits (approval / change / inference / transaction) 2) Receipt emitted (non-blocking) with stable reference + context refs 3) Receipt stored append-only in customer-controlled immutable storage 4) Later: receipts retrieved and verified for audit / dispute / investigation
